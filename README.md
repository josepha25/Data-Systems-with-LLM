# Data Systems with LLM Interface

A system where you can load a CSV file and then ask questions about the data 
in plain english. It uses Claude AI to turn your question into a SQL query 
and runs it against a SQLite database.

## What it does

You load a CSV, and then you can either write SQL directly or just ask a 
question like "show me all engineers" and it figures out the SQL for you.

There are 6 modules that each do one thing:

- **CSV Loader** - takes a CSV file and puts the data into the database
- **Schema Manager** - keeps track of what tables and columns exist
- **SQL Validator** - makes sure queries are safe before running them (SELECT only, no deleting or dropping tables)
- **Query Service** - the middleman between the interface and the database
- **LLM Adapter** - sends your question to Claude and gets SQL back
- **CLI** - the interface you actually type into

The CLI never talks to the database directly, everything goes through 
the Query Service. And the LLM output is always validated before 
anything gets executed.

cd Data-Systems-with-LLM
python3 -m venv venv
source venv/bin/activate

## Design choices

- The CLI has no direct database access on purpose — everything goes 
  through the Query Service so the system stays modular and testable
- SQL Validator runs on every query, even ones generated by the LLM, 
  because you can't fully trust AI output
- Tests were written before the implementation for the SQL Validator 
  (that's how we caught edge cases early)
- Schema is inferred automatically from the CSV so you don't have to 
  define it manually
